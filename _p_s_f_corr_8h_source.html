00064     struct HSMParams {
00065     /*
00066      * @brief Parameters that determine how the moments/shape estimation routines make
00067      *        speed/accuracy tradeoff decisions.
00068      *
00069      * @param nsig_rg            A parameter used to optimize convolutions by cutting off the galaxy
00070      *                           profile.  In the first step of the re-Gaussianization method of PSF
00071      *                           correction, a Gaussian approximation to the pre-seeing galaxy is
00072      *                           calculated. If re-Gaussianization is called with the flag 0x4 (as
00073      *                           is the default), then this approximation is cut off at nsig_rg
00074      *                           sigma to save computation time in convolutions.
00075      * @param nsig_rg2           A parameter used to optimize convolutions by cutting off the PSF
00076      *                           residual profile.  In the re-Gaussianization method of PSF
00077      *                           correction, a "PSF residual" (the difference between the true PSF
00078      *                           and its best-fit Gaussian approximation) is constructed. If
00079      *                           re-Gaussianization is called with the flag 0x8 (as is the default),
00080      *                           then this PSF residual is cut off at nsig_rg2 sigma to save
00081      *                           computation time in convolutions.
00082      * @param max_moment_nsig2   A parameter for optimizing calculations of adaptive moments by
00083      *                           cutting off profiles. This parameter is used to decide how many
00084      *                           sigma^2 into the Gaussian adaptive moment to extend the moment
00085      *                           calculation, with the weight being defined as 0 beyond this point.
00086      *                           i.e., if max_moment_nsig2 is set to 25, then the Gaussian is
00087      *                           extended to (r^2/sigma^2)=25, with proper accounting for elliptical
00088      *                           geometry.  If this parameter is set to some very large number, then
00089      *                           the weight is never set to zero and the exponential function is
00090      *                           always called. Note: GalSim script devel/modules/test_mom_timing.py
00091      *                           was used to choose a value of 25 as being optimal, in that for the
00092      *                           cases that were tested, the speedups were typically factors of
00093      *                           several, but the results of moments and shear estimation were
00094      *                           changed by <10^-5.  Not all possible cases were checked, and so for
00095      *                           use of this code for unusual cases, we recommend that users check
00096      *                           that this value does not affect accuracy, and/or set it to some
00097      *                           large value to completely disable this optimization.
00098      * @param regauss_too_small  A parameter for how strictly the re-Gaussianization code treats
00099      *                           small galaxies. If this parameter is 1, then the re-Gaussianization
00100      *                           code does not impose a cut on the apparent resolution before trying
00101      *                           to measure the PSF-corrected shape of the galaxy; if 0, then it is
00102      *                           stricter.  Using the default value of 1 prevents the
00103      *                           re-Gaussianization PSF correction from completely failing at the
00104      *                           beginning, before trying to do PSF correction, due to the crudest
00105      *                           possible PSF correction (Gaussian approximation) suggesting that
00106      *                           the galaxy is very small.  This could happen for some usable
00107      *                           galaxies particularly when they have very non-Gaussian surface
00108      *                           brightness profiles -- for example, if there's a prominent bulge
00109      *                           that the adaptive moments attempt to fit, ignoring a more
00110      *                           extended disk.  Setting a value of 1 is useful for keeping galaxies
00111      *                           that would have failed for that reason.  If they later turn out to
00112      *                           be too small to really use, this will be reflected in the final
00113      *                           estimate of the resolution factor, and they can be rejected after
00114      *                           the fact.
00115      * @param adapt_order        The order to which circular adaptive moments should be calculated
00116      *                           for KSB method. This parameter only affects calculations using the
00117      *                           KSB method of PSF correction.  Warning: deviating from default
00118      *                           value of 2 results in code running more slowly, and results have
00119      *                           not been significantly tested.
00120      * @param max_mom2_iter      Maximum number of iterations to use when calculating adaptive
00121      *                           moments.  This should be sufficient in nearly all situations, with
00122      *                           the possible exception being very flattened profiles.
00123      * @param num_iter_default   Number of iterations to report in the output ShapeData structure
00124      *                           when code fails to converge within max_mom2_iter iterations.
00125      * @param bound_correct_wt   Maximum shift in centroids and sigma between iterations for
00126      *                           adaptive moments.
00127      * @param max_amoment        Maximum value for adaptive second moments before throwing
00128      *                           exception.  Very large objects might require this value to be
00129      *                           increased.
00130      * @param max_ashift         Maximum allowed x / y centroid shift (units: pixels) between
00131      *                           successive iterations for adaptive moments before throwing
00132      *                           exception.
00133      * @param ksb_moments_max    Use moments up to ksb_moments_max order for KSB method of PSF
00134      *                           correction.
00135      * @param failed_moments     Value to report for ellipticities and resolution factor if shape
00136      *                           measurement fails.
00137      */
