44  struct HSMParams {
45  /*
46  * @brief Parameters that determine how the moments/shape estimation routines make
47  * speed/accuracy tradeoff decisions.
48  *
49  * @param nsig_rg A parameter used to optimize convolutions by cutting off the galaxy
50  * profile. In the first step of the re-Gaussianization method of PSF
51  * correction, a Gaussian approximation to the pre-seeing galaxy is
52  * calculated. If re-Gaussianization is called with the flag 0x4 (as
53  * is the default), then this approximation is cut off at nsig_rg
54  * sigma to save computation time in convolutions.
55  * @param nsig_rg2 A parameter used to optimize convolutions by cutting off the PSF
56  * residual profile. In the re-Gaussianization method of PSF
57  * correction, a "PSF residual" (the difference between the true PSF
58  * and its best-fit Gaussian approximation) is constructed. If
59  * re-Gaussianization is called with the flag 0x8 (as is the default),
60  * then this PSF residual is cut off at nsig_rg2 sigma to save
61  * computation time in convolutions.
62  * @param max_moment_nsig2 A parameter for optimizing calculations of adaptive moments by
63  * cutting off profiles. This parameter is used to decide how many
64  * sigma^2 into the Gaussian adaptive moment to extend the moment
65  * calculation, with the weight being defined as 0 beyond this point.
66  * i.e., if max_moment_nsig2 is set to 25, then the Gaussian is
67  * extended to (r^2/sigma^2)=25, with proper accounting for elliptical
68  * geometry. If this parameter is set to some very large number, then
69  * the weight is never set to zero and the exponential function is
70  * always called. Note: GalSim script devel/modules/test_mom_timing.py
71  * was used to choose a value of 25 as being optimal, in that for the
72  * cases that were tested, the speedups were typically factors of
73  * several, but the results of moments and shear estimation were
74  * changed by <10^-5. Not all possible cases were checked, and so for
75  * use of this code for unusual cases, we recommend that users check
76  * that this value does not affect accuracy, and/or set it to some
77  * large value to completely disable this optimization.
78  * @param regauss_too_small A parameter for how strictly the re-Gaussianization code treats
79  * small galaxies. If this parameter is 1, then the re-Gaussianization
80  * code does not impose a cut on the apparent resolution before trying
81  * to measure the PSF-corrected shape of the galaxy; if 0, then it is
82  * stricter. Using the default value of 1 prevents the
83  * re-Gaussianization PSF correction from completely failing at the
84  * beginning, before trying to do PSF correction, due to the crudest
85  * possible PSF correction (Gaussian approximation) suggesting that
86  * the galaxy is very small. This could happen for some usable
87  * galaxies particularly when they have very non-Gaussian surface
88  * brightness profiles -- for example, if there's a prominent bulge
89  * that the adaptive moments attempt to fit, ignoring a more
90  * extended disk. Setting a value of 1 is useful for keeping galaxies
91  * that would have failed for that reason. If they later turn out to
92  * be too small to really use, this will be reflected in the final
93  * estimate of the resolution factor, and they can be rejected after
94  * the fact.
95  * @param adapt_order The order to which circular adaptive moments should be calculated
96  * for KSB method. This parameter only affects calculations using the
97  * KSB method of PSF correction. Warning: deviating from default
98  * value of 2 results in code running more slowly, and results have
99  * not been significantly tested.
100  * @param max_mom2_iter Maximum number of iterations to use when calculating adaptive
101  * moments. This should be sufficient in nearly all situations, with
102  * the possible exception being very flattened profiles.
103  * @param num_iter_default Number of iterations to report in the output HSMShapeData structure
104  * when code fails to converge within max_mom2_iter iterations.
105  * @param bound_correct_wt Maximum shift in centroids and sigma between iterations for
106  * adaptive moments.
107  * @param max_amoment Maximum value for adaptive second moments before throwing
108  * exception. Very large objects might require this value to be
109  * increased.
110  * @param max_ashift Maximum allowed x / y centroid shift (units: pixels) between
111  * successive iterations for adaptive moments before throwing
112  * exception.
113  * @param ksb_moments_max Use moments up to ksb_moments_max order for KSB method of PSF
114  * correction.
115  * @param failed_moments Value to report for ellipticities and resolution factor if shape
116  * measurement fails.
117  */
