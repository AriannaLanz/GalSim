2 // Objects that make use of 2d FFT's in VERSION 3 of the FFTW package.
3 // I am ASSUMING that the FFTW is set up to do double-precision.
4 
5 // Notes:
6 // *All tables have even dimensions (enforced on construction).
7 //
8 // *The complex arrays (KTables) must be Hermitian, so transforms are real.
9 //
10 // *All arrays are 0-indexed.
11 //
12 // *FFTW arrays are stored in ROW-MAJOR order, meaning that in matrix notation, the most
13 // rapidly varying index is the last one. However in an "image" view of the array, we would
14 // label this as the x value that is increasing along rows. When doing real-to-complex
15 // transforms, the complex output contains (N/2+1) elements along this latter index, 0<=j<=N/2,
16 // and the j<0 elements must be obtained via conjugation. Imaginary parts of 0 and N/2 are zero.
17 //
18 // *This interface will assume an "image" based convention in which all access to real
19 // or complex elements is in (ix, iy) format, and ix will be the rapidly varying index, with
20 // the k-space arrays being half-sized in the x direction.
21 //
22 // **** so: when filling arrays, make ix your inner loop. And provide only kx>=0 to fill KTable.
23 //
24 // *xTable arrays have indices -N/2 <= ix,iy < N/2. To store in FFTW arrays, which assume
25 // 0<=j < N, we add N/2 to indices before accessing FFTW arrays. This means that k
26 // values need to be multiplied by -1^(ix+iy) before/after transforms.
27 //
28 // *kTable arrays can be accessed by -N/2 <= jx, jy <= N/2. FFTW puts DC at [0,0] element,
29 // so the code in this class changes negative input indices to wrap them properly, also
30 // considering that jx<0 must be conjugated.
31 //
32 // *"forward" transform, x->k, has -1 in exponent.

333  // Fill table from a function class:
334  template <class T>
335  void KTable::fill(const T& f)
336  {
337  clearCache(); // invalidate any stored interpolations
338  std::complex<double> *zptr=array;
339  double kx, ky;
340  for (int iy=0; iy< N/2; iy++) {
341  ky = iy*dk;
342  for (int ix=0; ix< N/2+1 ; ix++) {
343  kx = ix*dk;
344  *(zptr++) = f(kx,ky);
345  }
346  }
347  // wrap to the negative ky's
348  for (int iy=-N/2; iy< 0; iy++) {
349  ky = iy*dk;
350  for (int ix=0; ix< N/2+1 ; ix++) {
351  kx = ix*dk;
352  *(zptr++) = f(kx,ky);
353  }
354  }
355  return;
356  }
